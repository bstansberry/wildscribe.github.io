Attributes (22)

  • add-messageID-in-header If true, then the original message's message ID will be appended in the message sent to the destination in the header HORNETQ_BRIDGE_MSG_ID_LIST. If the message is bridged more than once, each message ID will be appended.
    • More Info
  • client-id The JMS client ID to use when creating/looking up the subscription if it is durable and the source destination is a topic.
    • More Info
  • failure-retry-interval The amount of time in milliseconds to wait between trying to recreate connections to the source or target servers when the bridge has detected they have failed.
    • More Info
  • max-batch-size The maximum number of messages to consume from the source destination before sending them in a batch to the target destination. Its value must >= 1.
    • More Info
  • max-batch-time The maximum number of milliseconds to wait before sending a batch to target, even if the number of messages consumed has not reached max-batch-size. Its value must be -1 to represent 'wait forever', or >= 1 to specify an actual time.
    • More Info
  • max-retries The number of times to attempt to recreate connections to the source or target servers when the bridge has detected they have failed. The bridge will give up after trying this number of times. -1 represents 'try forever'.
    • More Info
  • module The name of AS7 module containing the resources required to lookup source and target JMS resources.
    • More Info
  • paused Whether the JMS bridge is paused.
    • More Info
  • quality-of-service The desired quality of service mode (AT_MOST_ONCE, DUPLICATES_OK or ONCE_AND_ONLY_ONCE).
    • More Info
  • selector A JMS selector expression used for consuming messages from the source destination. Only messages that match the selector expression will be bridged from the source to the target destination.
    • More Info
  • source-connection-factory The name of the source connection factory to lookup on the source messaging server.
    • More Info
  • source-context he properties used to configure the source JNDI initial context.
    • More Info
  • source-destination The name of the source destination to lookup on the source messaging server.
    • More Info
  • source-password The password for creating the source connection.
    • More Info
  • source-user The name of the user for creating the source connection.
    • More Info
  • started Whether the JMS bridge is started.
    • More Info
  • subscription-name The name of the subscription if it is durable and the source destination is a topic.
    • More Info
  • target-connection-factory The name of the target connection factory to lookup on the target messaging server.

  • target-context The properties used to configure the target JNDI initial context.

  • target-destination The name of the target destination to lookup on the target messaging server.
