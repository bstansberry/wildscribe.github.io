JBoss AS7 7.0.0 Model Reference

  1. JBoss AS7 7.0.0

The root node of the server-level management model.

Children (9)

  • core-service Core services provided by the server.
    • service-container The central container that manages all services in a running standalone server or in a host controller in a management domain.
    • vault Security Vault for attributes.
    • server-environment The server environment.
    • module-loading The modular classloading system.
    • platform-mbean Provides the management interface for monitoring and management of the Java virtual machine as well as the operating system on which the Java virtual machine is running. Exposes the JDK-provided JMX MBeans in the java.lang, java.nio and java.util.logging JMX domains.
    • management The management services used to control a server or a host's host controller.
    • patching patch
  • deployment A list of deployments available for use on the server.
  • deployment-overlay A list of deployment overlays available for use by the server
  • extension A list of extension modules.
  • interface A list of named network interfaces available for use on the server.
  • path A list of named filesystem paths.
  • socket-binding-group A list of socket bindings available for use on the server.
  • subsystem The subsystems that make up the server's profile.
    • jaxrs The configuration of the JAX-RS subsystem.
    • jacorb The JacORB subsystem configuration.
    • io IO subsystem
    • jsf The JSF subsystem
    • jca The Java EE Connector Architecture (JCA) subsystem providing general configuration for resource adapters
    • jmx The configuration of the JMX subsystem.
    • threads The threading subsystem, used to declare manageable thread pools and resources.
    • webservices The configuration of the web services subsystem.
    • sar The SAR subsystem allowing you to deploy sar archives containing MBeans configured by META-INF/jboss-service.xml
    • remoting The configuration of the Remoting subsystem.
    • infinispan The configuration of the infinispan subsystem.
    • weld The configuration of the weld subsystem.
    • ejb3 The configuration of the ejb3 subsystem.
    • transactions The configuration of the transaction subsystem.
    • datasources The data-sources subsystem, used to declare JDBC data-sources
    • deployment-scanner An individual deployment scanner that scans a particular location
    • logging The configuration of the logging subsystem.
    • jdr JBoss Diagnostic Reporter (JDR) Subsystem.
    • undertow Undertow subsystem
    • messaging The messaging subsystem.
    • pojo Support JBoss5 and JBoss6 pojo configuration model.
    • jpa The configuration of the JPA subsystem.
    • naming The configuration of the naming subsystem.
    • ee The configuration of the EE subsystem.
    • mail The mail subsystem
    • jsr77 The configuration of the JSR-77 subsystem.
    • resource-adapters Configuration of the resource adapters subsystem.
    • security The configuration of the security subsystem.
  • system-property A list of system properties to set on the server.

Attributes (15)

  • launch-type The manner in which the server process was launched. Either "DOMAIN" for a domain mode server launched by a Host Controller, "STANDALONE" for a standalone server launched from the command line, or "EMBEDDED" for a standalone server launched as an embedded part of an application running in the same virtual machine.
    • More Info
  • management-major-version The major version of the JBoss AS management interface that is provided by this server.
    • More Info
  • management-micro-version The micro version of the JBoss AS management interface that is provided by this server.
    • More Info
  • management-minor-version The minor version of the JBoss AS management interface that is provided by this server.
    • More Info
  • name The name of this server. If not set, defaults to the runtime value of InetAddress.getLocalHost().getHostName().
    • More Info
  • namespaces Map of namespaces used in the configuration XML document, where keys are namespace prefixes and values are schema URIs.
    • More Info
  • process-type The type of process represented by this root resource. Always has a value of "Server" for a server resource.
    • More Info
  • product-name The name of the JBoss AS based product that is being run by this server.
    • More Info
  • product-version The version of the JBoss AS based product release that is being run by this server.
    • More Info
  • profile-name The name of the server's configuration profile.
    • More Info
  • release-codename The codename of the JBoss Application Server release this server is running.
    • More Info
  • release-version The version of the JBoss Application Server release this server is running.
    • More Info
  • running-mode The current running mode of the server. Either NORMAL (normal operations) or ADMIN_ONLY. An ADMIN_ONLY server will start any configured management interfaces and accept management requests, but will not start services used for handling end user requests.
    • More Info
  • schema-locations Map of locations of XML schemas used in the configuration XML document, where keys are schema URIs and values are locations where the schema can be found.
    • More Info
  • server-state The current state of the server controller; either STARTING, RUNNING or RESTART_REQUIRED
    • More Info

This is equivalent to an 'add', 'undeploy', 'deploy', 'remove' sequence where the new content has the same name as the content being replaced.
